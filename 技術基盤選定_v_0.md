# 技術基盤選定 v0.1

## 採用

- App: Next.js App Router
- UI: React + Tailwind CSS
- Contract: TypeScript + Zod
- Workspace: npm workspaces
- Shared packages:
  - `@ichijiuke/domain`
  - `@ichijiuke/ui`
  - `@ichijiuke/inquiry-engine`
  - `@ichijiuke/db`

## 理由

- 管理画面と公開チャットを 1 つの Web app にまとめられる
- route group で `auth / admin / public` を切りやすい
- 設定スキーマ、カテゴリ、UI primitives を package 化できるので 10 人並列でも衝突しにくい
- MVP 段階では server component 中心で進められ、過剰な client state を避けられる

## 次に載せるもの

- 認証: Auth.js
- DB: PostgreSQL + Drizzle ORM
- 通知: summary / urgent を分ける delivery layer

## 今回作ったもの

- `apps/web`: route skeleton
- `packages/domain`: 初期契約とカテゴリマスタ
- `packages/inquiry-engine`: stub 判定ロジック
- `packages/ui`: shell と表示部品
- `packages/db`: 初期 schema の受け皿
